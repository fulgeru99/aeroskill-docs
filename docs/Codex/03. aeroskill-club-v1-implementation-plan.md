# Prompt
define the exact V1 feature list as a buildable implementation plan: screens, admin modules, database entities, user flows, and priority levels.


Below is a buildable V1 implementation plan for **Aeroskill Club** as a polished membership club with a strong CRM foundation underneath.

**Priority Key**

- **P0:** Required for launch
- **P1:** Strong V1, highly recommended
- **P2:** Useful later, not needed for first launch

**V1 Product Goal**

Aeroskill Club V1 must let the club:

- Present a credible public aviation club website
- Convert visitors into paying members
- Manage member profiles, subscriptions, and member cards
- Publish and manage member benefits
- Track aviation partners, sponsors, contracts, and basic fleet records
- Communicate with members and partners from one CRM foundation

P1 entities:

- `CommunicationCampaign`
- `CommunicationRecipient`
- `AdminNote`
- `Task`
- `Aircraft`
- `AircraftDocument`
- `AuditLog`
- `Event`
- `NewsPost`

Important relationships:

- A `User` can have one `Member` profile.
- A `Member` has one active `Subscription`.
- A `Subscription` belongs to one `MembershipTier`.
- A `MemberCard` belongs to a `Member`.
- An `Organization` can have many `OrganizationTypes`.
- An `Organization` can have many `Contacts`.
- A `Benefit` belongs to a provider `Organization`.
- A `Benefit` can be available to multiple `MembershipTiers`.
- A `Contract` belongs to an `Organization`.
- A `Contract` can be linked to multiple `Benefits`.
- An `Aircraft` can be linked to an `Organization`, owner, or operator.
- A `Document` can attach to members, organizations, contracts, payments, or aircraft.

**Core User Flows**

P0 public-to-member flow:

1. Visitor lands on Home.
2. Opens Membership Tiers.
3. Chooses a tier.
4. Creates account.
5. Completes profile.
6. Pays subscription.
7. Lands in Member Dashboard.
8. Receives active digital member card.

P0 member renewal flow:

1. Member logs in.
2. Sees subscription status and renewal date.
3. Receives renewal reminder.
4. Pays renewal.
5. Member card validity updates automatically.

P0 benefit access flow:

1. Member logs in.
2. Opens Benefits Directory.
3. Filters by category.
4. Opens benefit detail.
5. Sees eligibility and redemption instructions.
6. Uses member card/QR where needed.

P0 admin member flow:

1. Admin opens Members.
2. Searches or filters by status/tier.
3. Opens member profile.
4. Reviews subscription, payments, notes, card status.
5. Updates status or adds admin note.

P0 CRM partner flow:

1. Admin creates Organization.
2. Adds organization types: sponsor, flight school, aerodrome, etc.
3. Adds contacts.
4. Adds contract.
5. Adds related benefits.
6. Publishes selected benefits or partner visibility.

P1 communication flow:

1. Admin creates announcement.
2. Selects audience: all members, tier, expiring members, partners.
3. Sends email.
4. System records delivery status.

**V1 Build Order**

1. Public website and membership tier content
2. Authentication and member onboarding
3. Subscription/payment flow
4. Member dashboard and digital card
5. Admin member/subscription/payment management
6. Organizations, contacts, benefits, contracts
7. QR validation and benefit eligibility
8. Communications and reminders
9. Fleet registry and documents
10. Audit log, tasks, and reporting polish

**Defer From V1**

- Aircraft booking
- Maintenance operations
- Native mobile app
- Partner self-service portal
- Advanced marketing automation
- Marketplace-style benefit redemption tracking
- Full event ticketing
- Complex multi-country tax/payment rules

The clean MVP test is: **can Aeroskill Club recruit a member, collect payment, issue a valid card, show benefits, and let admins manage the member and partner ecosystem from one place?** That is the right V1.
